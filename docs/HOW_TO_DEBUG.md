# 如何调试图片识别问题

## 快速开始

如果上传图片后没有显示数据,请按照以下步骤操作:

### 第1步: 打开开发者工具

**最简单的方法**: 按键盘上的 `F12` 键

**其他方法**:
- Windows/Linux: `Ctrl + Shift + I`
- Mac: `Cmd + Option + I`
- 或右键点击页面 → 选择"检查"

### 第2步: 切换到控制台

在开发者工具中,点击顶部的 **"Console"** (控制台)标签。

### 第3步: 上传图片

在系统中上传您的考试成绩截图。

### 第4步: 查看日志

上传后,控制台会显示详细的处理过程。

## 日志解读

### 看到这些日志说明正常:

```
=== OCR识别完成 ===
识别到 50 行文字
识别结果: ...
前200字符: 得分数据 68.8/100 ...

=== 开始解析OCR文本 ===
提取总分: 68.8 匹配结果: 68.8/100
提取用时: 396 秒, 匹配结果: 6分36秒
提取最高分: 100
提取平均分: 64
提取击败率: 73.3

--- 解析模块: 政治理论 ---
找到模块数据: ...
解析结果: 总题数=20, 答对=15, 正确率=75%, 用时=28秒

=== 解析完成 ===
总共解析到 25 个模块
```

### 常见问题

#### 问题1: 总分为0

**日志显示**:
```
提取总分: 0 匹配结果: undefined
```

**原因**: OCR没有识别到总分信息

**解决方法**:
1. 检查截图是否包含总分信息
2. 确保总分显示清晰
3. 尝试重新上传更清晰的截图

#### 问题2: 模块未解析

**日志显示**:
```
--- 解析模块: 政治理论 ---
未找到模块数据
```

**原因**: OCR识别的文字格式不匹配

**解决方法**:
1. 查看"OCR识别结果"中是否包含该模块
2. 检查模块名称是否完整
3. 确保截图包含该模块的完整信息

#### 问题3: 识别行数很少

**日志显示**:
```
识别到 5 行文字
```

**原因**: 图片质量不好或内容太少

**解决方法**:
1. 使用更高分辨率的截图
2. 确保截图清晰
3. 检查截图是否完整

## 需要帮助?

如果以上方法都无法解决问题:

1. **复制控制台日志**:
   - 在控制台中右键 → 选择"全选"
   - 右键 → 选择"复制"

2. **截图**:
   - 截取控制台日志的截图
   - 保存原始的考试成绩截图

3. **联系支持**:
   - 提供控制台日志
   - 提供原始截图
   - 描述问题现象
